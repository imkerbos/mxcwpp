**自保护**:
- Rocky 9: chattr +i 保护已生效
- CentOS 7: `/var/lib/mxsec/rules` 目录不存在，selfprotect 空转

### 2.4 Scanner（病毒扫描）— ❌ 阻塞性 BUG

**YARA-X 扫描**:
- 手动执行 `yr scan` 成功检测 EICAR
- 但插件解析结果为 0 — **JSON 格式不匹配**
- YARA-X v1.15.0 输出：`{"version":"1.15.0","matches":[{"rule":"...","file":"..."}]}`
- 代码期望：按行 NDJSON，字段名 `path`/`rules`/`identifier`

**ClamAV 扫描**:
- CentOS 7：`available=false`（glibc 2.17 太低），正确跳过
- Rocky 9：clamscan 加载病毒库 ~970MB RSS → cgroup 1G 限制 → **OOM kill → Agent 死循环重启**

**隔离箱**: API 正常但因无扫描结果而空

### 2.5 Remediation（漏洞修复）— ⚠️ 部分通过

- **CentOS 7**: 修复任务正确失败（EOL 源 `mirrorlist.centos.org` 不可用），诊断引擎输出合理提示
- **Rocky 9**: 任务创建正常（`dnf upgrade`），但因 Scanner OOM Agent 离线无法执行
- 安全机制完整：白名单/黑名单/shell 注入防护/预检/DryRun/审计日志
