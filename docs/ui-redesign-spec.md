## 一、整体布局规范

### 1.1 页面结构 (参考 Elkeid)

```
+----------------------------------------------------------+
|  NAVBAR (60px, 白色背景, 底部 1px 边框)                      |
|  [Logo 24px] [标题 18px] [版本号灰色药丸]    [语言] [用户菜单]  |
+----------+-----------------------------------------------+
|          |                                                |
|  SIDER   |   CONTENT AREA                                 |
| (200px)  |   - 页面标题 + 操作按钮                           |
|  浅色主题  |   - 统计卡片区                                    |
|  可折叠    |   - 筛选条件栏                                    |
|  手风琴    |   - 数据表格 / 内容区                              |
|          |                                                |
+----------+-----------------------------------------------+
```

### 1.2 与当前 MxSec 的关键差异

| 项目 | 当前 MxSec | Elkeid (目标) | 改动 |
|------|-----------|--------------|------|
| **侧边栏主题** | 暗色 (`#001529`) | 浅色 (白色背景 `#fff`, 右边框 `#E5E8EF`) | **改为浅色主题** |
| **侧边栏宽度** | 260px → 80px | 200px → 48px | **缩窄至 200px** |
| **Logo 位置** | 侧边栏顶部 | **顶部导航栏左侧** | **Logo 移到 Navbar** |
| **Navbar** | 仅右侧用户菜单 | 左侧 Logo+标题+版本, 右侧语言+用户 | **Navbar 增加 Logo** |
| **内容区背景** | `#f0f2f5` | `#f2f3f8` | 微调背景色 |
| **菜单模式** | inline, 多级可同时展开 | inline + **accordion** (同时仅展开一个) | **改为手风琴模式** |
| **菜单选中态** | 蓝色背景填充 | 左侧竖线指示器 (3px 蓝色竖条) | **改为竖线指示** |
| **内容包装** | 单个白色 card 包裹 `content-wrapper` | 页面内各区域独立卡片 | **去掉 content-wrapper, 改为独立卡片** |
| **最小宽度** | 无限制 | 1200px (`min-width`) | **设置最小宽度** |

### 1.3 布局参数

```
Navbar 高度:      60px
Sider 展开宽度:    200px
Sider 折叠宽度:    48px
Content 内边距:    24px
Content 背景色:    #f2f3f8
Card 圆角:        8px
Card 间距:        16px (gutter)
页面最小宽度:      1200px
```

- **手风琴模式**: 同一时间只展开一个子菜单

- **悬停效果**: `background-color: #F2F3F5` (浅灰)
- **展开子菜单背景**: 无特殊背景 (浅色主题下自然层次)
- **折叠态**: 仅显示图标, tooltip 提示文字
- **菜单项高度**: 一级 40px, 二级 36px
- **图标**: 使用 @ant-design/icons-vue, 与 Elkeid 风格对齐

## 九、代码架构优化

### 9.1 菜单配置数据化

当前 `BasicLayout.vue` 中菜单是硬编码模版 + 70 行 if/else-if watch。
参考 Elkeid 的 `menuConfig` 数组驱动模式:

```typescript
// ui/src/config/menu.ts
interface MenuItem {
  key: string
  title: string
  icon?: Component
  route?: string
  children?: MenuItem[]
  adminOnly?: boolean
}

export const menuConfig: MenuItem[] = [
  {
    key: 'dashboard',
    title: '安全概览',
    icon: DashboardOutlined,
    route: '/dashboard',
  },
  {
    key: 'assets',
    title: '资产中心',
    icon: DatabaseOutlined,
    children: [
      { key: 'hosts', title: '主机列表', route: '/hosts' },
      { key: 'asset-fingerprint', title: '资产指纹', route: '/asset-fingerprint' },
      { key: 'business-lines', title: '业务线管理', route: '/business-lines' },
    ],
  },
  // ...
]
```

用 `menuConfig` 驱动: 菜单渲染、路由映射、选中态计算, 消除重复代码。

### 9.2 去掉 content-wrapper

当前所有页面被 `.content-wrapper` 白色卡片包裹。Elkeid 风格是页面内各区域自带卡片:

```vue
<!-- 当前 (BasicLayout.vue) -->
<a-layout-content class="content">
  <div class="content-wrapper">  <!-- 去掉这层 -->
    <router-view />
  </div>
</a-layout-content>

<!-- 目标 -->
<a-layout-content class="content">
  <router-view />               <!-- 直接渲染, 页面自行管理卡片 -->
</a-layout-content>
```

每个页面自己用 `<a-card>` 或 `.mxsec-card` 包裹各区块。

### 9.3 减少 `!important` 覆盖

将 `global.less` 中的 Ant Design 覆盖尽量迁移到 `App.vue` 的 ConfigProvider token:

```typescript
// App.vue
const themeConfig = {
  token: {
    colorPrimary: '#165DFF',
    borderRadius: 4,
    colorBgContainer: '#FFFFFF',
    colorBgLayout: '#F2F3F5',
    // ...更多 token
  },
  components: {
    Table: { headerBg: '#FAFAFA', rowHoverBg: '#F7F8FA' },
    Card: { borderRadiusLG: 8 },
    // ...
  },
}
```

## 附录 A: Elkeid 技术栈对照

| 维度 | Elkeid | MxSec | 说明 |
|------|--------|-------|------|
| 框架 | React 18 | Vue 3 | 不同框架, 参考设计而非代码 |
| UI 库 | Arco Design | Ant Design Vue 4 | 组件 API 高度相似, 直接对应 |
| 状态管理 | Recoil | Pinia | 各自生态 |
| 路由 | React Router | Vue Router | 各自生态 |
| 构建 | Webpack | Vite | MxSec 更快 |
| 图表 | 自定义实现 | ECharts + vue-echarts | MxSec 更成熟 |
| 国际化 | 自定义 i18n | 无 (纯中文) | 暂不需要 |

Arco Design 与 Ant Design 的组件对应关系几乎 1:1, 设计语言高度一致, 可直接映射样式。
